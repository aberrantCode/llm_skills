Row-Level Security Patterns
Core Rule

Enable RLS on every user-data table. Policies default-deny — you must explicitly allow access.

ALTER TABLE public.profiles
  ENABLE ROW LEVEL SECURITY;

-- Must-have: define policies or nobody reads
Owner Access
CREATE POLICY "own records"
  ON posts FOR UPDATE
  USING (auth.uid() = user_id);
Admin via JWT Claim
USING (
  auth.jwt() ->> 'role'
    = 'admin'
);
Auth Trigger: Auto-create Profile on Signup
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, username)
  VALUES (NEW.id, NEW.email);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
Environment Variables
Local (.env.local)
SUPABASE_URL http://localhost:54321
SUPABASE_ANON_KEY <from supabase start>
DATABASE_URL postgresql://...@localhost:54322/postgres
Production (.env.production)
SUPABASE_URL https://xxxxx.supabase.co
SUPABASE_ANON_KEY <from dashboard> — safe for client
SUPABASE_SERVICE_ROLE_KEY <dashboard> NEVER expose — server only
DATABASE_URL ...pooler.supabase.com:6543/postgres?pgbouncer=true
Connection Pooling Note

Use ?pgbouncer=true on port 6543 for serverless (transaction mode). Port 5432 session mode is required for migrations and long-lived transactions.

Edge Functions
Basic Structure
// supabase/functions/hello/index.ts
import { serve } from 'https://deno.land/...';

serve(async (req) => {
  const { name } = await req.json();
  return new Response(
    JSON.stringify({ message: `Hello ${name}` }),
    { headers: { 'Content-Type': 'application/json' } }
  );
});
With Auth Context
const supabase = createClient(
  Deno.env.get('SUPABASE_URL'),
  Deno.env.get('SUPABASE_ANON_KEY'),
  {
    global: {
      headers: {
        Authorization: req.headers
          .get('Authorization')
      }
    }
  }
);
const { data: { user } }
  = await supabase.auth.getUser();
if (!user) return new Response(
  'Unauthorized', { status: 401 });

CLI Quick Reference
Lifecycle

Database

supabase link --project-ref Link to remote
supabase db pull Pull remote schema
Anti-Patterns
✗
Direct production changes — always use migrations; never touch production schema manually
✗
Disabled RLS — enable on all user-data tables; default-deny means you must write explicit policies
✗
Service role key in client code — never expose the service role key to browser or mobile; server-side only
✗
No connection pooling — always use PgBouncer (transaction mode) in serverless environments
✗
Skipping migration review — always check the generated SQL before running db reset or db push
✗
Committing .env files — add all .env* files to .gitignore immediately
