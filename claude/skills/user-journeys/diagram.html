User Journeys

User experience flows — journey mapping, UX validation, and error recovery. Tests real experiences, not just specifications.

"Specs test features. Journeys test experiences. A feature can pass all specs but still deliver a terrible experience."

Critical

Test every deploy
  • Signup to First Value
  • Checkout Purchase
  • Login to Dashboard

High

Test daily
  • Browse and Search
  • Update Profile
  • Invite Team Member

Medium

Test weekly
  • Weekly features
  • Secondary flows
  • Admin paths

Edge Cases

Test on change
  • Payment failure retry
  • Session timeout recovery
  • Offline reconnection

Journey Documentation Anatomy

Overview Attributes

  • Priority: Critical / High / Medium
  • User Type: New / Returning / Admin
  • Frequency: Daily / Weekly / One-time
  • Success Metric: conversion, time, drop-off

Per-Step Structure

  • User Action: what they do
  • System Response: what they see
  • Success Criteria: measurable checkboxes
  • Potential Friction: blockers + mitigations

Error Scenarios

  • Trigger: what causes the error
  • User Sees: exact error state
  • Recovery Path: back on track
  • Test: linked Playwright spec file

Clarity

  • User knows where they are
  • User knows next step
  • User knows what just happened

Speed

  • Page loads <2 seconds
  • Actions complete <3 seconds
  • Progress shown for longer ops

Forgiveness

  • Mistakes easy to undo
  • Errors explain what went wrong
  • Recovery path is clear

Accessibility

  • Keyboard navigation works
  • Screen reader announces changes
  • Focus management correct
  • Touch targets 44px+

Common Journey Patterns

Progressive Disclosure

  1. Show basic options only
  2. "Advanced" expands more options
  3. Expert mode unlocks everything

Guided Setup

  1. Welcome + single choice
  2. Core preference question
  3. Optional integrations (skippable)
  4. First action with guidance
  5. Success + remove training wheels

Recovery Journey

  1. Recognize returning user
  2. Restore previous state
  3. Acknowledge what happened
  4. Offer clear path forward
  5. Complete original goal

Journey Anti-Patterns — Avoid

Happy path only — test error recovery
Spec-driven — test user goals not features
Ignoring time — measure journey duration
Desktop-only — mobile journeys separately
No metrics — track completion and drop-off
Static journeys — update as behavior evolves
Skipping emotions — note frustration points
