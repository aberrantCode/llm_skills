Git Cleanup

audit stale worktrees + branches · merged-into-dev detection · safe deletion

Purpose
Identifies and removes stale git worktrees, local branches, and remote (origin) branches that have already been merged into dev. Protects anything with uncommitted local changes. Supports both squash-merge and standard-merge repositories.
Protected Always
dev · main · master · origin/dev · origin/main · any checked-out branch
Never Force Delete
Uses git branch -d (safe) not -D. If unexpectedly fails — log, do NOT escalate to force.
Requires dev Branch
If dev doesn't exist locally OR on origin → stop immediately. "Stale" cannot be defined without it.
executed as
Step-by-Step Workflow
  • 1
    Verify dev exists
    Check git branch --list dev AND git branch -r --list origin/dev. Both must exist.
  • 2
    Fetch + prune remote state
    git fetch origin --prune — ensures remote branch data is current before analysis.
  • 2b
    Detect merge strategy
    git log dev --merges --max-count=10. Empty → squash merges (use git cherry). Has merge commits → standard merges (use git branch --merged). Announce which was detected.
  • 3
    Collect + categorize worktrees
    Run git worktree list --porcelain. Skip main worktree. For each: check merged (per strategy) + check dirty (git -C <path> status --porcelain).
  • 4
    Collect + categorize local branches
    Exclude: dev, main, master, any branch checked out in a worktree. Apply merged check per detected strategy.
  • 5
    Collect remote branches (origin only)
    Exclude: origin/dev, origin/main, origin/master, origin/HEAD. Apply merged check per strategy.
  • 6
    Present stale list + ask user
    Display WORKTREES · LOCAL BRANCHES · REMOTE BRANCHES · BLOCKED (dirty) sections. Use AskUserQuestion: "Delete all listed, or exclude some?"
  • 7
    Execute deletions (in order)
    Worktrees first → local branches → remote branches. Track each result.
  • 8
    Report results
    Show ✓ successfully removed · ✗ failed (with error) · ⚠ skipped (uncommitted changes).
  • 9
    Update docs/git-log.md
    Create file if missing. Append rows to "Successfully Removed" · "Failed to Remove" · "Skipped" tables with date and scope.
Branch Categorization
STALE Merged into dev + clean (no uncommitted changes) Delete
BLOCKED Merged into dev + dirty (has uncommitted changes) Skip always
ACTIVE Not merged into dev (still has unshipped work) Ignore
Merge Strategy Detection
Standard Merges
git log dev --merges has output → use git merge-base --is-ancestor (exit 0 = merged). For local branches: git branch --merged dev.
Squash Merges
git log dev --merges empty → use git cherry dev <branch>. All lines start with - = merged. Any + line = has unshipped work → skip.
