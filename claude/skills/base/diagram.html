Architectural Patterns
Functional Core / Imperative Shell
Pure functions for business logic — no side effects, deterministic. Side effects only at boundaries (API, DB, FS).
Composition over Inheritance
No inheritance deeper than 1 level. Small, composable utilities. Dependency injection — pass deps, don't import directly.
Error Handling
Fail fast, fail loud. No silent failures. Every error is logged or thrown. Design APIs where misuse is impossible.

Quality Gates
Coverage Threshold
Minimum 80% code coverage. Business logic core aims for 100%. CI must fail below threshold.
Pre-Commit Hooks
All projects must run: linting (auto-fix), type checking, affected tests. Catches issues before CI.
Security Checks
No secrets in code. .env in .gitignore. Validate inputs with Zod. Parameterized queries only.
Testing Layers
Unit · Integration · E2E. Test behavior not implementation. Each test runs in isolation.
