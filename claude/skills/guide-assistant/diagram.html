Guide Assistant

step-by-step walkthroughs · interactive document execution · changelog tracking

Critical Rule
⚠
AskUserQuestion for EVERY prompt to the user. Never print a question as plain text and wait. Every prompt is a tool call. This rule has zero exceptions — not even "just a quick check".
One Step at a Time
Never present multiple steps together. Work through the document sequentially — never jump ahead.
Lazy File Creation
Don't create the notes file until it's actually needed. Never create files proactively "just in case".
Preserve the Document
Minimal edits only — checkboxes, inline notes, and changelog. Never reformat or rewrite content not requested.
executed in 3 phases
Phase 1 — Setup
1.1
Identify Source File
If no file path provided → AskUserQuestion. Then read with Read tool.
1.2
Check Checklist Format
Does document use - [ ] / - [x] checkboxes?
If NOT a checklist:
AConvert in-place (add checkboxes to source)
BCreate new checklist file alongside source
CKeep as-is, track mentally (no edits)
1.3
Ask About Value Capture
Should values (IPs, usernames, config settings) be recorded in notes?
  • Yes — record in source or notes file
  • No — only check steps off
  • Ask me each time — prompt per step
1.5
Changelog Setup
All edits to source file must be recorded in a ## Changelog appended at the very end.
  • Date · change description
  • Never silently modify files
Phase 2 — Step-by-Step Walkthrough
2.1
Explain the Step
What it involves · Why it matters · Prerequisites / warnings
2.2
Prompt via AskUserQuestion
Options presented:
✓Done — it worked
⚠Done — with issues
→Skip this step
✗Instructions seem wrong/outdated
2.3
On Completion
Change - [ ] to - [x]. Record any captured values. Add changelog entry if file was edited.
2.4 Issue Flow
If issues reported:
1. AskUserQuestion: describe what went wrong
2. Ask where to record: source file or notes file
3. Write issue + resolution to chosen location
4. Add changelog entry
5. Re-ask: complete or blocked?
2.5 Invalid/Outdated Instructions Flow
1. AskUserQuestion: confirm update needed
2. Ask for corrected instructions (or infer)
3. Edit source file in-place — replace outdated content
4. Add changelog entry with what changed and why
5. Continue from corrected step
Phase 3 — Completion
1. Summarize
Steps completed · Steps skipped · Issues noted and resolved
2. Report Locations
Tell user where notes or captured values were recorded
3. Final Changelog Entry
Walkthrough completed — all steps processed
Files Produced (Lazy Creation)
Source File (modified)
original.md — checkboxes added, steps checked off, inline notes, changelog section appended
Notes File (optional)
original-notes.md — only created when first needed: issues arise, values captured, etc.
Checklist File (optional)
original-checklist.md — only if user chose Option B (separate checklist)
