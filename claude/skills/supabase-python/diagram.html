SQLModel + Pydantic Patterns
SQLModel — src/db/models.py
class PostBase(SQLModel):
    title: str
    content: Optional[str] = None
    published: bool = False

class Post(PostBase, table=True):
    __tablename__ = "posts"
    id: UUID = Field(
        default_factory=uuid4,
        primary_key=True)
    author_id: UUID = Field(
        foreign_key="profiles.id")
    created_at: datetime = Field(
        default_factory=datetime.utcnow)

class PostCreate(PostBase): pass
class PostRead(PostBase):
    id: UUID; author_id: UUID
FastAPI Route Pattern
@router.post(
    "/",
    response_model=PostRead,
    status_code=201)
async def create_post(
    db: DbSession,
    user: CurrentUser,
    post_in: PostCreate,
):
    post = Post(
        **post_in.model_dump(),
        author_id=UUID(user.id),
    )
    db.add(post)
    await db.commit()
    await db.refresh(post)
    return post

uvicorn src.main:app --reload --port 8000
Anti-Patterns
✗
Using Supabase client for DB queries — use SQLAlchemy/SQLModel with asyncpg for all database access
✗
Sync database calls in async routes — use asyncpg; synchronous calls block the event loop
✗
Not closing sessions — use async with AsyncSessionLocal() or yield in dependency
✗
Missing auth dependency — always inject CurrentUser on protected routes; never skip JWT validation
✗
Blocking I/O in async handlers — use async libraries throughout; never call sync file I/O in an async context
