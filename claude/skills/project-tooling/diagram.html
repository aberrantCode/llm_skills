Supabase Setup
  • New project — supabase projects create → supabase link --project-ref
  • Local dev — supabase start / stop / db reset
  • Migrations — supabase migration new <name> → supabase db push
  • Types — supabase gen types typescript --local > src/types/database.ts
  • Pull remote schema — supabase db pull
  • Rule — always develop locally first, never direct prod DB changes
GitHub Repository Setup
  • Create & push — gh repo create <name> --private --source=. --push
  • Branch protection — require PR reviews + CI status checks on main
  • Secrets — gh secret set VERCEL_TOKEN, SUPABASE_ACCESS_TOKEN
  • Default branch — gh repo edit --default-branch main
  • Anti-pattern — no hardcoded secrets, no manual deploys, no direct prod changes

"db:types": "supabase gen types..."
Anti-patterns: No hardcoded secrets — use CLI env or GitHub secrets — automate deploys via CI/CD, never manual — always develop Supabase locally first — use migrations for schema changes, never direct DB edits — require branch protection (PR reviews + CI checks) — keep dev/staging/prod separate.
