02 — Locator Strategy (Priority Order)
✅✅
Role-based (BEST)
Mirrors user interaction. Accessible & resilient to DOM changes.
getByRole('button', { name: 'Submit' })
getByRole('textbox', { name: 'Email' })
✅
Label / Text / TestID
User-facing identifiers. Stable against visual redesigns.
getByLabel('Email')
getByTestId('submit-btn')
⚠️
CSS selectors (AVOID)
Brittle — breaks when styles or classes change.
locator('.btn-primary')
❌
XPath (NEVER)
Extremely brittle. Breaks on any structural DOM change.
locator('//div[@class]/button[1]')

04 — Web-First Assertions (Auto-Wait)
👁 Visibility
toBeVisible()
toBeHidden()
toBeAttached()
📝 Text Content
toHaveText('exact')
toContainText('part')
toHaveValue('input')
🎛 State
toBeEnabled()
toBeDisabled()
toBeChecked()
toBeFocused()
🌐 Page
toHaveURL('/dash')
toHaveTitle(/title/)
toHaveScreenshot('pg.png')
❌ Never Use
waitForTimeout(3000)
Manual waits cause flaky tests
05 — Dead Link Detection (REQUIRED)
🔗 Dead Link Detection REQUIRED on every project

Every project MUST include dead link detection tests. Run these on every deployment. A broken link = broken user experience.

1.
Navigate to each page in PAGES_TO_CHECK array
2.
Collect all a[href] elements from the page
3.
Skip mailto:, tel:, and anchor links
4.
HTTP HEAD each URL, verify response.ok()
5.
Fail test with specific dead link URL + found-on page
6.
Also check broken images with img[src] validation
06 — Anti-Patterns to Avoid
🚫
Hardcoded waits — waitForTimeout() makes tests flaky. Use auto-waiting assertions.
🚫
CSS/XPath selectors — brittle. Use role/text/testid.
🚫
Shared state between tests — each test must be fully isolated.
🚫
Missing awaits — use ESLint rule no-floating-promises.
🚫
Testing third-party sites — mock external dependencies.
🚫
Huge test files — split by feature or page.
