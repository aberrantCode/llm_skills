ship-to-dev

Automates the complete feature branch → squash merge → DEV workflow. From uncommitted work (or a pre-committed branch) all the way to a synced local DEV — with mandatory test, coverage, and lint gates at every step.

Inputs & Outputs
Inputs
Uncommitted local changes or committed feature branch
Branch name ($BRANCH) and commit message ($MSG)
All tests passing at >= 80% coverage
Zero new lint / type errors
Outputs
Squash-merged commit on remote DEV branch
Feature branch deleted (local + remote)
Local DEV synced to origin/dev
Timing report appended to logs/timing.jsonl
Step-by-Step Pipeline (never skip, never reorder)
0
Detect Context Auto
Resolve REPO_ROOT. Detect if in secondary worktree (IN_WORKTREE). Check if current branch already has commits ahead of dev — if so, set AHEAD>0 and skip steps 1–3.
1
Ask for Branch & Commit Message Skip if pre-committed
Use AskUserQuestion to collect $BRANCH and $MSG. Infer 2 suggestions each from diff and recent commits. If user already provided both, use directly.
2
Pull Latest (only if behind) Skip if pre-committed / BEHIND=0
git fetch origin → check BEHIND count. If 0, skip entirely. If >0: stash → git pull --rebase → pop stash. Handle conflicts via AskUserQuestion.
3
Stage All Changes Skip if pre-committed
git add --all. Show git status --short and git diff --cached --stat. If staging area is empty → stop, nothing to ship.
4
Test / Coverage / Lint Gate MANDATORY — never skip
Runs for all branches — pre-committed or new. Five sub-steps — each can STOP the workflow.
4A: Identify changed files 4B: Full test suite → STOP if fail 4C: Test files exist → STOP if missing 4D: ≥80% coverage → STOP if below 4E: Lint + type + build → STOP if new errors
5
Create Feature Branch & Commit Auto
git checkout -b $BRANCH then git commit -m "$MSG". Verify with git log --oneline -1.
6
Push to Origin Auto
git push -u origin $BRANCH
7
Create PR Targeting DEV Auto
gh pr create --base dev --head $BRANCH --title "$MSG".
If capability IDs found in commits → include a Capabilities table. If PR already exists → reuse it, continue to Step 8.
8
Merge PR (Squash) From REPO_ROOT
cd $REPO_ROOT first. Stash working tree if dirty. If in worktree → remove worktree before merge (prevents branch-lock error). gh pr merge $BRANCH --squash --delete-branch --subject "$MSG"
9/10
Cleanup & Sync DEV Final
Conditional checkout dev → conditional delete local branch (-d, or -D if merged) → prune worktree if needed → git pull origin dev → restore stash → print timing report.
Step 4 Gate — 5 Sub-Steps (all must pass)
4A
Identify Changed Files
Categorize by stack: API_SRC, WEB_SRC, EXT_SRC. Skip stacks with no source changes.
4B — STOP
Full Test Suite
Run pwsh Start-Tests.ps1 -NoPrompt -Parallel -SkipE2E. Non-zero exit → STOP, report failures.
4C — STOP
Test File Existence
For each changed source file, verify test file exists. Missing → STOP, list them. Must create before retrying.
4D — STOP
≥80% Coverage
Per-file coverage for changed sources only. Python via pytest + coverage.json. TS via vitest coverage. Below 80% → STOP.
4E — STOP
Lint / Type / Build
Ruff (API), ESLint + tsc + Vite build (Web), tsc (Extension). Zero new errors. Pre-existing debt exceptions documented in CLAUDE.md.
Key Decision Points
Condition Check Outcome
Branch ahead of dev CURRENT_BRANCH ≠ dev AND AHEAD > 0 SKIP steps 1–3
Behind remote BEHIND = 0 SKIP pull step
Nothing staged git diff --cached empty STOP — nothing to ship
Tests fail Start-Tests.ps1 exit ≠ 0 STOP — fix failures first
Missing test file test -f <expected-path> fails STOP — create test file first
Coverage below 80% per-file pct < 80 STOP — add tests, retry
New lint errors ruff/eslint/tsc non-zero outside known debt STOP — fix errors first
PR already exists gh pr create exits 1: "already exists" CONTINUE — reuse existing PR URL
Error Recovery
Rebase conflict
Run git rebase --abort. Stop and tell user. Never force through a conflict.
Push rejected
Run git pull --rebase origin $BRANCH then retry push. Do not force-push.
PR merge fails (status checks)
Show gh pr checks $BRANCH. Do NOT force merge. Fix the failing check.
Branch used by worktree
Step 8 removes worktree before merge. If still fails: git worktree prune && rm -rf $WORKTREE_PATH
local changes overwritten
Step 8 stashes before merge. If PR already merged: check with gh pr view before retrying.
gh not authenticated
Run gh auth login and pause until complete. No gh commands without auth.
Integration
Requires: gh CLI authenticated
Requires: dev branch on remote
Uses: pwsh Start-Tests.ps1
Outputs to: logs/timing.jsonl
Pairs with: using-git-worktrees
