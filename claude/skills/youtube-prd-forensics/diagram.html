2
Download Video
Download at up to 1080p using yt-dlp with best video + audio merge.
yt-dlp -f "bv*[height<=1080]+ba/b[height<=1080]"
-o "docs/<BASENAME>_reference_1080p.%(ext)s" <URL>
3
Extract YT Artifacts
Pulls metadata JSON, description text, and comments JSON without re-downloading video.
yt-dlp --skip-download --write-info-json
--write-description --write-comments
4
Transcript / Subtitles
Try YouTube's auto-subtitles first (VTT format). Fall back to Whisper/whisper.cpp generation from local video if unavailable.
yt-dlp --write-auto-subs --write-subs
--sub-langs "en.*" --sub-format "vtt"
5
Keyframe Extraction
Extract one frame every 5 seconds and build a timestamped index for the analysis phase. Temporary — deleted at the end.
extract-keyframes-5s.ps1
-VideoPath <file> -OutputDir docs/images/video_frames_5s
6
Analysis Phase
Cross-reference keyframes and transcript together to derive requirements. Distinguish observed facts from inferences. Identify UI controls, data flows, layer toggles, and failure states.
8
PRD.md Generation
Write requirements tied to explicit timestamps, embed screenshots per section, include source metadata. Append change-summary table when editing existing PRD.
📁 Required Persisted Artifacts (kept after keyframe cleanup)
docs/<project>_reference_1080p.*
Downloaded video source at up to 1080p resolution
docs/transcript_notes.md
Curated transcript with timestamps
docs/PRD.md
Final requirements document with embedded images
docs/images/screenshots/*
Selected full-frame or cropped screenshots used by PRD
docs/youtube_metadata.json
Video metadata, title, creator, publish date, duration
docs/youtube_comments.json
Comment data for notable insights and feedback
PRD Content Requirements
Source metadata (URL, title, creator, date, duration)
Requirements mapped to explicit timestamps
Screenshots embedded under each major section
Facts vs inferred requirements clearly labeled
UI controls, data sources, layer toggles captured
Failure states and workflow behavior documented
Notable comments included in metadata section
Change-summary table on edits to existing PRD
