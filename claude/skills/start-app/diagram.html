SKILL / START-APP

Start App

End-to-end workflow for discovering, selecting, and executing the correct startup procedure for any modern application — then validating success and recovering from failures across every major stack.

1
Discover Existing Scripts
Search scripts/ for filenames starting with start, run, execute, or launch. Also check for docker-compose.yml, Makefile, package.json, and Procfile. Build a candidate list.
2
Match Candidates to Intent
Use the user's prompt to filter by technology name, component name, or environment. Then branch on how many candidates were found.

CASE A — Not found

Offer to create a startup script, run directly, or accept manual command. Generate scripts/Start-App.ps1 using stack detection if user agrees.

CASE B — Exactly one

Read the script, check for parameter overrides implied by the user's prompt, then execute directly.

CASE C — Multiple

Present all candidates via AskUserQuestion with one-line summaries. After selection, offer default vs. custom parameters.

3
Execute and Inspect Results
Run the command, capture stdout/stderr, stream for 10–20 seconds and evaluate the tail. Classify as Success or Failure from output patterns.
# Success indicators
Listening on | Running on | ready | compiled | serving | ✓ | port number

# Failure indicators
Error | EADDRINUSE | failed | ModuleNotFoundError | exited with code
⚡ Step 3a — Failure Recovery (up to 3 auto-fix retries)
Error PatternLikely CauseSuggested Fix
EADDRINUSE Port occupied by another process Kill process on that port or use a different port
ModuleNotFoundError Missing dependencies npm install / pip install -r requirements.txt / uv sync
command not found Missing runtime (node, python, docker) Install the required runtime
connection refused Database/service not running Start the required dependency first
.env not found Missing environment config Copy from .env.example or set the missing variable
Cannot connect to Docker daemon Docker not running Start Docker Desktop
