03 — Severity Levels & Gate Policy
🔴 Critical
Must fix immediately. Vulnerabilities, injections, exposed secrets.
❌ NO COMMIT
🟠 High
Should fix before commit. Auth gaps, memory leaks, race conditions.
❌ NO COMMIT
🟡 Medium
Fix soon. N+1 queries, missing pagination, code clarity issues.
✅ CAN COMMIT
🟢 Low
Nice to have. Naming, extraction suggestions, minor improvements.
✅ CAN COMMIT
ℹ️ Info
Informational only. Suggestions with no urgency.
✅ CAN COMMIT

07 — Review Checklists

💾 Before Every Commit

Run /code-review on staged changes
No critical (🔴) issues remaining
No high (🟠) issues remaining
Security concerns addressed
Performance issues considered

🔀 Before Every PR

Full review of all changes vs base
All critical/high issues resolved
Tests added for new functionality
Documentation updated if needed

🚀 Before Every Deployment

Final review of deployment diff
Security scan passed
No new vulnerabilities introduced
Rollback plan documented
