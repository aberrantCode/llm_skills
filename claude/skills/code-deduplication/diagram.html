Check Before You Write

Prevent semantic code duplication by maintaining a capability index. Always search for existing functionality before implementing something new — AI doesn't copy-paste, it reimplements.

01 — Core Philosophy

The Deduplication Mantra

The problem isn't duplicate code — it's duplicate PURPOSE. Before writing any new function:

1
Check CODE_INDEX.md for existing capabilities
2
Search codebase for similar functionality with grep/ripgrep
3
Extend existing code if possible (add optional params)
4
Only create new if nothing suitable exists — then update index
02 — Decision Tree

04 — Common Duplication Patterns
❌ Reimplementing existing utility
Creating validateEmail() when isEmail() already exists in the index.
function validateEmail(email) {
return /^[^\s@]+@[^\s@]+/.test(email);
}
✅ Check index, use existing
Search CODE_INDEX.md → Found isEmail() → import and use.
import { isEmail } from '@/utils/validate';

if (isEmail(userInput)) { ... }
❌ Scattered inline validation
Same email check copy-pasted across signup.ts, profile.ts, invite.ts, settings.ts, admin.ts.
// In 5 different files:
if (!email || !email.includes('@') ...) {..}
✅ Extract once, import everywhere
Define in utils/validate.ts, update index, import in all files.
// utils/validate.ts
export const isEmail = (e: string) =>
e?.includes('@') && e.length >= 5;
