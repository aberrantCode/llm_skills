web-design-guidelines skill

Web Interface
Guidelines Review

Reviews UI code against Vercel's Web Interface Guidelines — fetches live rules from source, checks specified files, and outputs findings in terse file:line format.

Author
Vercel
v1.0.0
01

Receive Trigger

User invokes with file pattern or asks to review UI / check accessibility / audit design

02

Fetch Guidelines

WebFetch retrieves latest rules from vercel-labs/web-interface-guidelines on GitHub

03

Read Target Files

Read specified files or ask user which files/pattern to review if none provided

04

Output Findings

Check against all fetched rules, output in terse file:line format specified in guidelines

Trigger Phrases

"review my UI" "check accessibility" "audit design" "review UX" "check my site against best practices" /web-design-guidelines src/ /web-design-guidelines *.tsx

Live Guidelines Source — Always Fresh

https://raw.githubusercontent.com/vercel-labs/web-interface-guidelines/main/command.md
Fetched before each review — never cached, always latest rules
Output format is defined inside the fetched guidelines themselves
If no files specified, ask user for target files or pattern

Output Format — Terse file:line

src/components/Button.tsx:42 missing aria-label on icon button src/pages/Home.tsx:118 contrast ratio 2.6:1 fails WCAG AA src/styles/globals.css:77 outline: none without focus-visible ring # Format defined in fetched guidelines # Claude applies all fetched rules to # each specified file, outputs each # violation as file:line message

Typical Rule Categories Checked

  • Color contrast ratios (WCAG AA/AAA)
  • Missing ARIA labels on interactive elements
  • Focus indicator removal without replacement
  • Touch target sizes below minimum
  • Semantic HTML structure violations
  • Keyboard navigation gaps
  • Missing alt text on images
  • Responsive layout breakpoints
  • Animation/motion safety (prefers-reduced-motion)

Full Review — Component Libraries

  • Component libraries with multiple developers
  • Design system enforcement
  • Pre-launch accessibility audit
  • Frequent UI changes requiring regression checks
  • Use with Playwright for automated CI coverage

Quick Check — Solo / MVP

  • Pre-PR UI review on specific changed files
  • Spot check a single component
  • Verify after dark mode implementation
  • Confirm accessibility on new form
  • Manual checklist + this skill = sufficient for MVP
