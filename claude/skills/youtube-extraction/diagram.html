/recreate-files — pipeline

orchestrator → 7 sub-skills → reconstructed files on disk

The thin command /recreate-files takes a YouTube url and a free-text request. If either is missing, the parent skill uses AskUserQuestion before proceeding. Once both parameters are resolved, seven sub-skills run in order — acquiring the video, transcript, and comments; extracting frames; recognising file content via multimodal vision; stitching frames into canonical files using line-level LCS without duplication; and emitting a markdown report. Only the working frames/ directory is deleted at cleanup.

Stitching without duplicates. Scrolling IDE panes produce overlapping frames where each line appears in N consecutive captures. file-reconstruction merges using a line-level longest common subsequence alignment: for each new frame, it finds the contiguous tail of the accumulated buffer that matches a contiguous head of the new frame, then appends only what's after the overlap. Verbatim quotes in transcript.md and creator code blocks in comments.json fill gaps where the camera cut past unread lines.
