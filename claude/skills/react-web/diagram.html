Component Patterns

How Components Are Built

Functional Components Only
  • Always define explicit Props interface
  • No class components
  • Return type: JSX.Element
  • Default props in destructuring
Extract Logic to Hooks
  • useXxx.ts holds all logic
  • Page components are pure presentation
  • Hooks are independently testable
  • renderHook() for hook tests
CSS Modules (Preferred)
  • Button.module.css per component
  • Tailwind as alternative
  • Never inline style objects
  • Extract repeated combos
Forms: React Hook Form + Zod
  • zodResolver for validation
  • Type-safe schema inference
  • Error messages per field
  • No uncontrolled surprises
State Management

The Escalation Ladder

01 Local State useState Start here — always
02 Server State React Query Fetching, caching, invalidation
03 Global UI State Zustand Only when prop drilling > 2 levels
React Query — Server State
queryKey for cache identity · staleTime to avoid over-fetching · invalidateQueries on mutation success · Never fetch in useEffect
Zustand — Global UI State
Immutable updates via set() · setUser, toggleTheme actions · No reducers · Persist middleware for localStorage
Anti-Patterns

What to Avoid

Inline functions in JSX
Use useCallback for handlers
Logic in render
Extract to custom hooks
Prop drilling > 2 levels
Use context or Zustand
useEffect for derived state
Use useMemo instead
