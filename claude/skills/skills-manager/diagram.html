Operations Reference — All 9 Commands
/find-skills [path]
read-only
Discover new or changed skills across the workstation. Never writes anything.
?
path given? scan that dir : AskUserQuestion confirm → scan all
✕
Skip installed-from: llm_skills files
!
status: deprecated → always Deprecated (ignores diff)
≡
Classify: New | Changed | Unchanged | Orphan | Deprecated
≈
Note version: delta if archive differs
⌖
Detect companion commands for New/Changed skills
▶
Output structured Find Report
/sync-skill [name]
mutating
Archive a skill including its full bundle. Resolves source conflicts and generates a diagram.
?
name given? search sources : find-skills scan → AskUserQuestion confirm all
⊕
Conflict resolution: global > this repo > other projects (alpha)
⌖
Detect companion commands + sub-skills
▶
Write SKILL.md, commands/, sub-skills/ to archive
◈
Invoke visual-explainer:generate-web-diagram → save diagram.html
▶
Update SKILL.md ## Diagram section + README row
/install-skill [name] [target]
mutating
Deploy a skill bundle from archive into a project. Resolves dependencies via requires: and injects the installed-from marker.
?
name given? locate bundle : enumerate not-installed → AskUserQuestion confirm
?
target-dir? use it : AskUserQuestion for path
⟳
Resolve requires: list recursively; deduplicate
◇
AskUserQuestion: confirm N files to write
▶
Write SKILL.md + inject installed-from: llm_skills; sub-skills with marker; commands without marker
/update-skill [name]
mutating
Update installed skills in the current project to match newer archive versions. Preserves the installed-from marker.
⌖
Scan project's .claude/skills/ for installed-from: llm_skills
≈
Diff each against archive; skip if identical
◇
AskUserQuestion: confirm with version delta (v1.0→v1.2)
▶
Overwrite SKILL.md preserving marker; update commands + sub-skills
◈
Regenerate diagram for each updated skill
/import-skill [name]
mutating — reverse
Import project-level skill changes back into the archive and global profile. The only reverse-flow operation.
?
name given? locate in project : scan non-installed changed → AskUserQuestion confirm
!
installed-from guard: force-import warning (default Cancel)
≈
Show diff summary (lines added / removed)
◇
AskUserQuestion: "Yes, import both" | "Archive only" | Cancel
▶
Write to archive + user profile; bundle new companion commands
/push-skill [name]
mutating
Push a skill bundle from the archive to the global user profile (~/.claude/). No installed-from marker is added — global profile is a source tier.
?
name given? push that bundle : enumerate all → AskUserQuestion confirm
!
status: draft → skip with warning (drafts not pushed)
?
~/.claude/skills/<name> exists & differs? → AskUserQuestion overwrite/skip
▶
Write SKILL.md, sub-skills, commands to ~/.claude/ (no marker)
/search-skill [query]
read-only
Full-text search across all archived skills. Ranked by field priority. Never writes anything.
?
query given? use it : AskUserQuestion "What are you looking for?"
⌖
Search: name > description > headings > body
≡
Rank: exact name > desc hit > heading hit > body; ties alphabetical
▶
Output ranked Search Results with diagram links
/audit-skills
read-only
Comprehensive health check. No confirmation needed — no changes made.
1
Missing diagrams: skills lacking diagram.html
2
README broken links: row targets that don't exist
3
README missing rows: skill dirs with no README entry
4
Bad frontmatter: missing name: or description:
5
Deprecated installs: projects using status: deprecated skills
6
Parity gaps: Claude-only skills (no Codex/Gemini port)
▶
Output structured Audit Report
/backfill-diagrams [name]
mutating
Retroactively generate diagram.html for all archived skills missing one.
?
name given? check that skill : scan all claude/skills/*/ for missing diagram.html
◇
AskUserQuestion: confirm N skills to generate
◈
Invoke visual-explainer:generate-web-diagram → save diagram.html
▶
Update SKILL.md Diagram sections + README diagram links

Archive Invariants — Never Violate
1. Never delete from archive. Copies are permanent. Deletions are manual curatorial decisions only.
2. Flow is source → archive — except /import-skill, the only explicit reverse path.
3. README is the authoritative index. Every archived skill has a row; every row points to a real file.
4. Toolset by directory, not content. claude/, codex/, gemini/ determine the toolset column.
5. installed-from gating. Installed copies are not project-developed skills — never synced back without force.
