SKILL / SKILLS-MANAGER

Skills Manager

Full lifecycle management of LLM skills across the workstation — nine operations covering discovery, archiving, deployment, updating, and health-checking.

Archive
Single source of truth at C:\development\llm_skills. Consolidates all skills from global profile and all projects.
Skill Bundle
SKILL.md + sub-skills/ + commands/ — all travel together on install/push.
Installed Copy
Marked installed-from: llm_skills in frontmatter. Skipped by find and sync — not a project-developed skill.
Source Priority
Global profile → this repo → other projects (alphabetical). Highest priority source wins on conflict.
🔍
/find-skills
Read-only discovery. Classifies each SKILL.md as New, Changed, Unchanged, Orphan, or Deprecated.
  • Skip installed-from copies
  • Compare vs archive (diff)
  • Detect companion commands
  • Outputs structured report
⬆
/sync-skill
Archive a skill bundle: SKILL.md + commands + sub-skills + diagram + README update.
  • Resolve highest-priority source
  • Write full bundle to archive
  • Generate diagram.html
  • Update README row
⬇
/install-skill
Deploy a skill bundle from archive into a project, resolving all dependencies first.
  • Resolve requires: dependencies
  • Inject installed-from marker
  • Write sub-skills + commands
  • AskUserQuestion confirms
🔄
/update-skill
Update installed skills when archive has newer versions. Shows version delta before confirming.
  • Scan for installed-from marker
  • Diff vs archive version
  • Confirm with version delta
  • Regenerate diagram on update
📥
/import-skill
Reverse flow: push project-developed changes back to archive and global profile.
  • Reject installed-from copies
  • Show diff (lines added/removed)
  • Write to archive + ~/.claude
  • Bundle new companion commands
🌐
/push-skill
Push archive skills to global profile so they're available in all projects.
  • Skip draft status skills
  • Confirm overwrite if differs
  • No installed-from marker added
  • Writes sub-skills + commands
🗂
/search-skill
Keyword search across all archived skills, ranked by match quality.
  • name match → highest priority
  • description → high priority
  • section headings → medium
  • body text → lower
🏥
/audit-skills
Read-only health check: missing diagrams, README drift, bad frontmatter, deprecated installs, parity gaps.
  • Missing diagram.html files
  • README broken/missing rows
  • Deprecated skill installs
  • Claude-only parity gaps
🖼
/backfill-diagrams
Retroactively generate diagram.html for archived skills that don't have one yet.
  • Find skills lacking diagram.html
  • Generate via visual-explainer
  • Add ## Diagram section
  • Update README diagram links
Archive Invariants — Never Violate
Never delete from archive Flow: source → archive (except /import-skill) README is authoritative index Toolset by directory, not content installed-from marker gates project vs archive All user interaction via AskUserQuestion
