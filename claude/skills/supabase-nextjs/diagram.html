Supabase + Next.js

App Router patterns with Supabase Auth, Drizzle ORM, server components, server actions, and realtime subscriptions.

Next.js App Router Drizzle ORM Supabase Auth SSR
Core Principle

Drizzle for queries, Supabase for auth/storage, server components by default.

Use Drizzle ORM for type-safe database access. Use Supabase client only for auth, storage, and realtime. Prefer server components — use client components only when needed for interactivity.

Supabase Client Variants
01
Browser Client
createBrowserClient()
Client components only. Uses @supabase/ssr. For auth state, OAuth redirects, file uploads, realtime.
02
Server Client
createServerClient()
Server components, server actions, route handlers. Reads/writes cookies for session. Must await cookies() in Next.js 15+.
03
Middleware Client
updateSession()
Runs in middleware to refresh session token. Must return the response with updated cookies — never skip this step.
Server Actions & Auth Helpers
Auth Helpers — src/lib/auth.ts
export async function requireAuth() {
  const supabase = await createClient();
  const { data: { user } }
    = await supabase.auth.getUser();
  if (!user) redirect('/login');
  return user;
}

export async function requireGuest() {
  const user = await getUser();
  if (user) redirect('/dashboard');
}
Server Action — posts.ts
'use server';

export async function createPost(fd) {
  const user = await requireAuth();
  const [post] = await db
    .insert(posts)
    .values({
      authorId: user.id,
      title: fd.get('title'),
    })
    .returning();
  revalidatePath('/dashboard');
  redirect(`/posts/${post.id}`);
}
Drizzle Schema — src/db/schema.ts
export const profiles = pgTable('profiles', {
  id:        uuid('id').primaryKey(),  // References auth.users
  email:     text('email').notNull(),
  name:      text('name'),
  avatarUrl: text('avatar_url'),
  createdAt: timestamp('created_at').defaultNow().notNull(),
});

export const posts = pgTable('posts', {
  id:        uuid('id').primaryKey().defaultRandom(),
  authorId:  uuid('author_id').references(() => profiles.id).notNull(),
  title:     text('title').notNull(),
  published: boolean('published').default(false),
  createdAt: timestamp('created_at').defaultNow().notNull(),
});

// Inferred types — always use these
export type Profile = typeof profiles.$inferSelect;
export type Post    = typeof posts.$inferSelect;

    { event: '*', table: 'posts' },
    (payload) => {
      // INSERT / UPDATE / DELETE
      setPosts(prev => [...]);
    }

Anti-Patterns
✗
Using Supabase client for DB queries — use Drizzle for all type-safe database access
✗
Fetching in client components — prefer server components; they render on the server with direct DB access
✗
Not using middleware for auth — session refresh via middleware is critical; without it tokens expire silently
✗
Calling cookies() synchronously — must await cookies() in Next.js 15+
✗
Missing revalidatePath — always revalidate after mutations or stale data persists in the cache
✗
Service key in client code — NEXT_PUBLIC_ variables are exposed to the browser; service role key must stay server-only
