Key Principles
↻
Summarize after every phase
Brief "here's what I captured" before moving on catches misunderstandings before they compound.
🔍
Phase 3 scan is essential
Actively scan — don't skip. Use findings in Phase 5 with specific file names, not abstractions.
📐
Push back on vague ACs
"It should work" is not an acceptance criterion. Press for something measurable or observable.
🚫
Never skip phases
Each phase builds context for the next. Skipping any creates implementation debt and misalignment.
✂️
Out of Scope is valuable
One of the most important sections — prevents scope creep and aligns stakeholders before coding starts.
📌
Lean on codebase scan
Suggest specific affected files from Phase 3 rather than speaking in abstractions. Ground the spec in reality.
