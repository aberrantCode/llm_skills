⚡ SKILL / REMOTE-INSTALLER

Remote PowerShell Installer

Domain expertise for install.ps1 and the in-app self-update block — covering auto-elevation, GitHub Releases API, safe download-before-delete ordering, .env backup/merge, and module installation.

⚠ Critical Prerequisite: GitHub Releases must be published with gh release create. The /releases/latest API returns 404 if only git tags exist — never assume a tag equals a release.
1
install.ps1 — Repo Root Bootstrap
S1 Auto-elevation — re-launches via Start-Process pwsh -Verb RunAs, never sudo
S2 Resolve latest release — GitHub API /releases/latest wrapped in try/catch → exit 0 on network failure
S3 Compare versions — reads version.txt; skips install if already up to date, jumps to run prompt
S4 Backup .env into memory — BEFORE any directory deletion; content held in $envBackup
S5 Download ZIP first — Invoke-WebRequest to TEMP; on failure leaves old install intact and exits 1
S6 Remove old, extract new — Remove-Item only AFTER successful download; detect extracted root dynamically
S7 Write version.txt — after successful extract, records installed version string
S8 Restore & merge .env — additive-only algorithm: backup values always win; new keys from .env.example added transparently
S9 Install PS module — only if Get-Module -ListAvailable returns empty; omit block if no module needed
S10 Run prompt — gated by $env:REPO_SKIP_RUN_PROMPT so auto-updates don't double-prompt
2
Update-Check Block — In-App Script
P1 Placement: immediately after elevation guard, before module imports or UI setup
P2 Dev-safe guard: entire block wrapped in if (Test-Path $versionFile) — dev/clone environments skipped; no network calls
P3 Version compare — reads local version.txt, calls GitHub API with 5s timeout; silently skips if offline
P4 Set skip-run-prompt — sets $env:REPO_SKIP_RUN_PROMPT=1 before invoking installer; clears after
P5 Re-launch & return — starts new version via Start-Script; return exits stale instance
P6 PSScriptRoot depth: adjust ..\..\ segments to match actual script depth in repo
🛡 Safeguards Checklist — verify before declaring complete
⬇ Download-before-deleteZIP downloaded to TEMP before Remove-Item on install dir
💾 .env backup firstRead into $envBackup before any directory removal
🔍 Dynamic root detectionGet-ChildItem -Directory | Select-Object -First 1; never hardcode zip folder name
🌐 Offline = exit 0API unreachable → clean exit; being offline is not an error
