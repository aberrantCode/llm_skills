02 — Key Code Patterns
❌ JWT in localStorage (XSS vulnerable)
localStorage.setItem('token', token)
// Any XSS attack can steal this token
✅ JWT in httpOnly cookie
res.setHeader('Set-Cookie',
`token=${token};
HttpOnly; Secure;
SameSite=Strict`)
❌ Secret in env var exposed to client
// Visible in browser bundle!
const k = import.meta.env.VITE_SECRET_KEY
✅ Secret stays server-side
// In API route / server function only:
const apiKey = process.env.SECRET_KEY
if (!apiKey) throw new Error('...')
❌ Logging sensitive data
console.log('Login:', { email, password })
console.log('Card:', { cardNumber, cvv })
✅ Redacted logging
console.log('Login:', { email, userId })
console.log('Pay:', { last4: card.last4 })
03 — Automated Security Tests
AUTH
Unauthenticated access blocked
Request protected routes without a token.
expect(response.status).toBe(401)
AUTHZ
Insufficient role blocked
Request admin route with a non-admin user token.
expect(response.status).toBe(403)
VALIDATION
Invalid input rejected
Send malformed or invalid data to API endpoints.
expect(response.status).toBe(400)
RATE LIMIT
Rate limiting enforced
Send 101 requests in rapid succession to a rate-limited endpoint.
expect(tooManyRequests.length).toBeGreaterThan(0) // 429
04 — Pre-Deployment Security Checklist
🚀 Before ANY production deployment
Secrets: No hardcoded secrets, all in env vars
Input Validation: All user inputs validated
SQL Injection: All queries parameterized
XSS: User content sanitized with DOMPurify
CSRF: Protection tokens enabled on mutations
Auth: Proper httpOnly token handling
Authorization: Role checks on sensitive ops
Rate Limiting: Enabled on all API endpoints
HTTPS: Enforced in production environment
Security Headers: CSP, X-Frame-Options set
Error Handling: No sensitive data in responses
Logging: No secrets or PII in logs
Dependencies: Up to date, no CVEs
Row Level Security: Enabled in Supabase
CORS: Restricted to known origin allowlist
File Uploads: Validated (size, type, extension)
