type patterns
Discriminated Union — Result<T>
{ ok: true; value: T } | { ok: false; error: string }
Type-safe error handling without exceptions. Narrows correctly in both branches.
Branded Types — IDs
type UserId = string & { readonly brand: unique symbol }
Prevents accidentally passing a UserId where an OrderId is expected.
Const Assertions — Literals
['pending','active','closed'] as const
Derive a union type from an array: type Status = typeof STATUSES[number]
Zod — Runtime Validation
z.object({ email: z.string().email() })
Schema validates at runtime; z.infer<> derives the static TypeScript type.
anti-patterns — never use these
✕any type — use unknown and narrow
✕Type assertions (as) — use type guards
✕Non-null assertions (!) — handle null explicitly
✕@ts-ignore without explanation
✕Enums — use const objects or union types
✕Classes for data — use interfaces/types
✕Default exports — use named exports
