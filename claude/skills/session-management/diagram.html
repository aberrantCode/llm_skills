Session Management

context preservation · tiered summarization · resumability

Core Principle
Checkpoint at natural breakpoints, resume instantly. Long development sessions risk context loss. Proactively document state, decisions, and progress so any session can resume exactly where it left off — whether returning after a break or hitting context limits.
governed by
Tiered Summarization Rules
TIER 1 — Quick Update
current-state.md only
Trigger: After any small task or todo item
  • Update Active Task
  • Update Progress
  • Update Next Steps
  • ~30 seconds to complete
TIER 2 — Full Checkpoint
current-state.md + decisions.md
Trigger: Feature complete · Any decision · ~20 tool calls · Context switch
  • Update full current-state.md
  • Log decisions to decisions.md
  • Update files being modified table
TIER 3 — Session Archive
archive/ + full checkpoint
Trigger: End of session · Major milestone · ~50+ tool calls
  • Create archive/YYYY-MM-DD.md
  • Full checkpoint first
  • Clear verbose current-state.md
  • Update code-landmarks.md

Session Start Protocol
1. Read current-state.md
Announce what was found: "Resuming from: [last state]". Never start blind.
2. Check active.md todos
Review what's in-progress or blocked. Understand pending work.
3. Continue from "Next Steps"
Confirm next steps before proceeding. Check recent decisions if needed.
Session End Protocol
1. Create archive entry
Archive to session/archive/YYYY-MM-DD.md. Include tasks done, decisions, code changes.
2. Update current-state.md
Full checkpoint with handoff notes. Clear verbose exploration paths.
3. Specific next steps
Next steps must be specific and actionable — not vague. Next session resumes instantly.
Anti-Patterns
  • ⚠ No state tracking — flying blind, can't resume
  • ⚠ Overly verbose state — keep it scannable, not a novel
  • ⚠ Stale state files — update regularly or they're useless
  • ⚠ Missing decisions — future you won't remember why
  • ⚠ No code landmarks — wastes time re-discovering codebase
  • ⚠ Never archiving — session files become cluttered
  • ⚠ Skipping checkpoint after decisions — key context lost
  • ⚠ No handoff at session end — next session starts blind
