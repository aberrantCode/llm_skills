patterns
Pydantic — Data Validation
BaseModel with Field constraints.
Validates at runtime; type annotation is the schema. Use for request/response DTOs, config, and any external data boundary.
Dependency Injection
Never import db or services directly in business logic — pass them as parameters. get_user(user_id, db: Database) instead of importing a global. Enables unit testing without patching.
Result Pattern
Core functions return Result[T] dataclass with value | None and error | None. No exceptions in core layer. Exceptions are infrastructure concerns only.
anti-patterns — never use these
✕from module import *
✕Mutable default arguments (def f(x=[]))
✕Bare except: clauses
✕# type: ignore without explanation
✕Global variables for state
✕Classes when plain functions suffice
