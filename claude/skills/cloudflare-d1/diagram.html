Serverless SQLite at the edge — Workers runtime, Drizzle ORM for type-safe queries, Wrangler CLI, and migration workflow via version control.

D1 Stack
D1 Database
Serverless SQLite
SQLite at the edge. Zero cold starts. Replicated globally. Up to 10GB, free tier: 5M reads/day, 100K writes/day.
Workers
Edge Runtime
V8 isolates, not Node.js. Access D1 via env.DB binding. Hono for routing (optional).
Drizzle + Kit
Type-safe ORM
Schema-first type generation. Drizzle Kit generates SQL migrations. Use drizzle(d1, { schema }).

Schema & Worker Implementation
src/db/schema.ts
export const users = sqliteTable('users', {
  id: integer('id')
    .primaryKey({ autoIncrement: true }),
  email: text('email').notNull().unique(),
  name: text('name').notNull(),
  role: text('role', {
    enum: ['user', 'admin']
  }).default('user'),
  createdAt: text('created_at')
    .default(sql`CURRENT_TIMESTAMP`)
});

// Inferred types
export type User =
  typeof users.$inferSelect;
export type NewUser =
  typeof users.$inferInsert;
src/index.ts (Hono)
const app = new Hono<{
  Bindings: { DB: D1Database }
}>();

app.use('*', async (c, next) => {
  c.set('db', createDb(c.env.DB));
  await next();
});

app.post('/users', async (c) => {
  const db = c.get('db');
  const body = await c.req.json();
  const result = await db
    .insert(users)
    .values(body)
    .returning();
  return c.json(result[0], 201);
});

Anti-Patterns
✗
Single large database — design for multiple smaller D1 databases (per-tenant); D1 is horizontally scaled
✗
No migrations — always version control schema changes; use Drizzle Kit + Wrangler migrations, not ad-hoc SQL
✗
Raw SQL everywhere — use Drizzle for type safety; raw strings have no compile-time column checking
✗
Large blobs in D1 — use R2 for file storage; D1 row size limit is 1MB
✗
Complex joins — D1 is SQLite; keep queries simple; complex joins may be better served by a relational DB
✗
Ignoring free tier limits — monitor usage; 100K writes/day on free tier is easy to exceed in production
