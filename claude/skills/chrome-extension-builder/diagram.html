Build Workflow

Four Steps to Production

1
Gather Requirements
Determine which components are needed: background service worker (always), content script, side panel, popup, options page. Identify target sites and required permissions.
2
Create Project Structure
Run pnpm dlx wxt@latest init <name> --template react. Organize entrypoints/, content/, lib/, and components/.
3
Configure WXT + Manifest
Set permissions, host_permissions, and optional_permissions in wxt.config.ts. Add React module. Configure manifest fields.
4
Implement Components
Background handler, content scripts with defineContentScript(), React UIs in entrypoints/. Use Shadow DOM for content script UI isolation.

Common Patterns

Key Implementation Patterns

Storage Wrapper Typed WXT storage with per-doc keys: local:doc:{id}. Get, set, and watch methods. Isolates storage key management from business logic.
Message Protocol Union type MessageType for all cross-context messages. PROTOCOL_VERSION for compatibility. sendMessage() with generic return type for type-safe responses.
Shadow DOM UI createShadowRootUi() isolates CSS from host page. React root mounted inside shadow. Prevents style leakage into host page.
Native Messaging NativeAdapter interface with connect/send/onMessage/disconnect. Mock adapter for development (USE_MOCK_NATIVE=true). Swap adapters via environment variable.
Text Insertion insertText() via Selection API for standard editors. insertViaClipboard() for Google Docs workaround (execCommand paste). Returns boolean for success/failure.
