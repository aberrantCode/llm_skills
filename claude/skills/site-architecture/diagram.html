Site Architecture

site-architecture skill — robots.txt, sitemap, meta tags, Core Web Vitals, AI crawlers

Philosophy
Content is king. Architecture is the kingdom.
Great content buried in poor architecture won't be discovered. This skill covers the technical foundation that makes your content findable by Google, Bing (traditional search), GPTBot, ClaudeBot, PerplexityBot (AI assistants), and social platforms (Open Graph, Twitter Cards).

Homepage: 1.0 · Pricing: 0.9 · About: 0.8 · Contact: 0.7 · Blog posts: 0.8 · Tags/categories: 0.6
Meta Tags
Essential Tags
charset, viewport, title (50–60 chars), description (150–160 chars), canonical, lang, robots.
Open Graph
og:type, og:url, og:title, og:description, og:image (1200×630), og:site_name. Article: +published_time, modified_time.
Twitter Cards
summary_large_image card type. twitter:site, twitter:creator, title, description, image.
URL Structure
  • Lowercase only — /blog/my-post not /Blog/My-Post
  • Hyphens not underscores — /my-page not /my_page
  • No trailing slashes — configure 301 redirect
  • Descriptive slugs — /pricing not /p
  • No query params for content — /blog/title not ?id=123
  • Max 3–4 levels deep — /blog/category/post
Canonical & Redirects
  • Every page needs a canonical — even if it's the primary URL
  • UTM params — canonical to clean URL (?utm_source stripped)
  • HTTP → HTTPS — 301 redirect + canonical to HTTPS
  • www vs non-www — pick one, canonical everywhere
  • Pagination — canonical page 2+ to page 1 or use rel=prev/next
core web vitals
Core Web Vitals Targets
LCP — Largest Contentful Paint
≤ 2.5s
≤ 4.0s needs work
> 4.0s poor
INP — Interaction to Next Paint
≤ 200ms
≤ 500ms needs work
> 500ms poor
CLS — Cumulative Layout Shift
≤ 0.1
≤ 0.25 needs work
> 0.25 poor
LCP Fixes
  • Optimize largest image (WebP)
  • Preload critical assets
  • CDN for static assets
  • Enable gzip/brotli compression
  • Minimize render-blocking JS
INP Fixes
  • Minimize JS execution time
  • Break up long tasks
  • Web workers for heavy compute
  • Optimize event handlers
  • Lazy load non-critical JS
CLS Fixes
  • Set dimensions on images/videos
  • Reserve space for dynamic content
  • Avoid inserting content above fold
  • Use transform for animations
  • Preload fonts with display: swap
Security Headers
  • Strict-Transport-Security — HSTS, includeSubDomains, preload
  • X-Frame-Options — SAMEORIGIN (prevent clickjacking)
  • X-Content-Type-Options — nosniff
  • Referrer-Policy — strict-origin-when-cross-origin
  • Permissions-Policy — restrict camera, mic, geolocation
Verification & Submission
  • Google Search Console — verify via DNS TXT or HTML file
  • Submit sitemap — GSC → Sitemaps → Add /sitemap.xml
  • Bing Webmaster — submit sitemap separately
  • Rich Results Test — validate structured data
  • robots.txt tester — use Google's built-in tester
Required public files: public/robots.txt or app/robots.ts — public/sitemap.xml or app/sitemap.ts — public/favicon.ico — public/og-image.jpg (1200×630) — public/apple-touch-icon.png (180×180) — public/manifest.webmanifest.
