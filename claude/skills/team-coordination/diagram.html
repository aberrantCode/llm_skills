Team Coordination

multi-person projects · shared state · todo claiming · conflict prevention · handoffs

Core Philosophy
Multiple devs, multiple Claude sessions, one codebase. Coordination > Speed. Communication > Assumptions. Before you start: check who's working on what. Before you claim: make sure nobody else has it. Before you decide: check if it's already decided. Before you push: pull and sync state.

Contributor Status Icons
🟢 Active — Currently working
🟡 Paused — Stepped away, will return
🔴 Blocked — Needs help / waiting
⚪ Offline — Not working today
Claim Format
**Claimed:** @handle (YYYY-MM-DD HH:MM TZ)

Add to todo in active.md + update state.md + commit + push.
Start Session Checklist
1
git pull origin main
2
Read team/state.md — who's active, what's claimed
3
Check claimed todos — don't take what's claimed
4
Claim your todo in active.md
5
Update state.md with your session info
6
Push state changes before starting work
7
Start working 🚀
End Session Checklist
1
Commit your work (even if WIP)
2
Update current-state.md
3
Update team state.md → Paused or Done
4
If passing to someone: create handoff note
5
Unclaim todo if abandoning it
6
Push everything 📤
Conflict Prevention
File checkBefore modifying a file, check state.md for who's touching what — coordinate if conflicting
Pre-pushgit pull origin main — resolve conflicts before pushing
PR taggingAlways tag area owners in PRs — authentication owner, payments owner, etc.
DecisionsPull latest decisions.md before deciding. If similar decision exists, follow it — consistency > preference
