domain error hierarchy
DomainError extends Error
message: string
code: string (readonly)
statusCode: number = 400
NotFoundError
statusCode: 404
code: NOT_FOUND
ValidationError
statusCode: 400
code: VALIDATION_ERROR
Global middleware catches DomainError, ZodError, and unknown errors — formats each to { error: { code, message } }.

testing strategy
U
Unit — mock the repo with jest.fn(), test service logic in isolation. No DB, no HTTP.
I
Integration — real test DB, supertest. beforeAll creates test app + DB, afterAll destroys.
%

validated config (Zod)
1
Define ConfigSchema with z.object({ NODE_ENV, PORT, DATABASE_URL, API_KEY })
2
Call ConfigSchema.parse(process.env) at startup
3
Zod throws immediately if any env var is missing or wrong type — fail fast, never silently

✕Blocking the event loop

✕console.log in production
