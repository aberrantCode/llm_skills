credentials skill

Centralized API Key Management

Securely load API keys from a centralized access file (e.g., ~/Documents/Access.txt). Parse, validate, and configure project environments automatically.

01 — Project Setup Workflow
1
Ask for Credentials File
Ask the user: "Do you have a centralized access keys file? Provide the path (e.g., ~/Documents/Access.txt) or type 'manual'."
Default locations: ~/Documents/Access.txt · ~/Access.txt · ~/.secrets/keys.txt
2
Parse the File
Use regex pattern matching to extract known key formats regardless of file format (colon, key=value, or informal). Show masked preview.
sk-ant-*** → ANTHROPIC_API_KEY | rnd_*** → RENDER_API_KEY
3
Validate Keys
Make HTTP calls to each service API to confirm the key is active. Show ✓ or ✗ for each credential found.
✓ ANTHROPIC_API_KEY: Valid ✗ SUPABASE_URL: Not found
4
Create .env File
Write found keys to project .env file. Add .env to .gitignore if not already present. Never commit Access.txt path to git.
# Auto-generated from ~/Documents/Access.txt
5
Report Missing Keys
List any required credentials not found with links to where they can be obtained (Supabase dashboard, API console, etc.).

04 — Security Rules
🚫
NEVER commit Access.txt
Never commit the credentials file or its path to git. It belongs only on your local machine.
🙈
NEVER log full keys
Always mask keys: show first 8 + "..." + last 4 chars only when displaying to the user.
✅
ALWAYS .gitignore .env
Add .env to .gitignore immediately. Never commit environment files to source control.
🔒
VALIDATE before use
Test each key with a lightweight API call to confirm it's valid and active before configuring the project.
