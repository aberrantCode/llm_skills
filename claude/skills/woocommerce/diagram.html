WooCommerce REST API

Integrate with WooCommerce stores — products, orders, customers, webhooks. Base URL: https://store.com/wp-json/wc/v3/

Prerequisites & Auth Setup
Store Requirements
  • WordPress + WooCommerce plugin
  • HTTPS enabled (required)
  • Permalinks: Post name (not Plain)
API Key Generation
  • WooCommerce → Settings → Advanced → REST API
  • Add key → Set permissions (Read/Write)
  • Copy Consumer Key & Secret (shown once)
Resource Operations
Products
  • GET products (with filters)
  • GET products/:id
  • POST products (create)
  • PUT products/:id (update)
  • DEL products/:id
  • GET products/:id/variations
Orders
  • GET orders (status/date filters)
  • GET orders/:id
  • POST orders (create)
  • PUT orders/:id (update status)
  • POST orders/:id/notes
Customers
  • GET customers (by email)
  • POST customers (create)
  • PUT customers/:id
Misc
  • POST coupons
  • GET products/categories
  • GET reports/sales
  • GET reports/top_sellers
  • POST webhooks

Verify webhook signatures via X-WC-Webhook-Signature header — HMAC-SHA256 of payload using your webhook secret.
Pagination & Headers
Response headers for paginating large datasets:
X-WP-Total: total number of items
X-WP-TotalPages: total number of pages
Use ?page=N&per_page=100 — max 100 per page. Loop until page >= totalPages.
Error Handling — Status Codes

Checklists
Before Integration
  • WooCommerce plugin installed
  • HTTPS enabled on store
  • Permalinks: Post name
  • API keys generated
  • Webhook secret set
Security
  • Keys in env variables
  • Webhook signatures verified
  • HTTPS for all API calls
  • Rate limiting handled
Anti-Patterns
  • Plain permalinks break API
  • HTTP in production
  • Large single requests (paginate)
  • Keys hardcoded in code
  • Skipping webhook verification
