06 — Security in CI/CD
🔒
Always use read-only sandbox in CI/CD to prevent file modifications.
--sandbox read-only
🗑️
Drop sudo in public repos to prevent API key self-reading attacks.
--safety-strategy drop-sudo
🔑
Use GitHub Secrets for API key. Never hardcode or echo keys in logs.
${{ secrets.OPENAI_API_KEY }}
📅
Run on PRs only (not every commit) to manage costs and API rate limits effectively.
