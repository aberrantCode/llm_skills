Universal Task Router

Inspects any repo, detects its project-management convention, reads or builds a backlog, prioritises with a weighted heuristic, and hands the user-chosen task to the right specialist agent. Caches to docs/what-next.md so follow-up runs skip re-analysis.

Nine-Step Pipeline
01 · CACHE

Check docs/what-next.md

Fingerprint match → fast path. Mismatch → re-analyse. File absent → cold start.

02 · STRUCTURE

Analyse repo layout

Detect stacks, monorepo markers, and meaningful folders that become area prefixes.

03 · PM FRAMEWORK

Detect the convention

File-based probes in priority order — project-manager wins over backlog.md wins over TODO.md.

04 · TASK STORE

Locate or bootstrap

Found a file → use it. Missing → AskUserQuestion: /project-manager, backlog.md, code analysis, or manual input.

05 · COLLECT

Classify tasks

Walk the checklist. Categorise as pending, in-progress, blocked, or completed.

06 · PRIORITISE

Weighted heuristic

+40 blocks others · +35 security · +25 failing tests · +20 in-progress · priority tag · age bonus.

07 · PRESENT

AskUserQuestion: top three

Always the AskUserQuestion tool — never prose prompts. User can pick or request the full backlog.

08 · HAND OFF

Spawn specialist agent

Map keywords → planner, tdd-guide, code-reviewer, security-reviewer, architect, etc.

09 · PERSIST

Refresh cache

Update fingerprints, area map, current_task. Archive completed items to backlog-archive.md.
