SKILL / PUBLISH-GITHUB

Publish to GitHub

End-to-end workflow: secrets detection, .gitignore hardening, main/dev branch setup, and GitHub branch protection rules — every time, without shortcuts.

PHASE 1
Prerequisites
Check and install required tools before touching the repo. Hard-abort if gh is missing or unauthenticated.
gh --version gh auth status gitleaks version ABORT if gh missing WARN if gitleaks missing
PHASE 2
Git Initialization
Initialize fresh repo or adapt existing one. Ensures default branch is main. Runs gitleaks history scan on existing repos.
⑂ No .git/ → git init --initial-branch=main  |  .git/ exists → rename master→main if needed, then scan history
PHASE 3
Project Files
Create security and normalization files. Existing files are respected — only missing ones are created.
.gitignore (created if missing) .gitattributes (eol=lf) .git/hooks/pre-commit (gitleaks)
PHASE 4
Visibility
Infer from user's original request. Only prompts via AskUserQuestion if "public" or "private" was not stated.
Public Private
PHASE 5
Initial Commit
Verify git identity, ensure on main branch. Stage and commit if repo has no commits yet; skip if commits already exist.
git config user.email / user.name git add -A && git commit
PHASE 6
Create Repo & Push main
Single gh repo create command creates the GitHub repo, sets origin, and pushes main atomically.
gh repo create <name> --source=. --push
PHASE 7
Create dev Branch
Create and push the integration branch.
git checkout -b dev git push -u origin dev
PHASE 8
Branch Protection
Configure protection rules via GitHub REST API. Falls back gracefully for free-plan private repos (returns manual setup instructions).
⑂
main — requires PR + 1 reviewer, stale reviews dismissed, no direct push
dev — requires PR, 0 approvals (self-merge OK); fallback to 1 if API rejects 0
403 → manual setup instructions (free plan) 422 → fallback to 1 reviewer on dev
PHASE 9–10
Switch & Summary
Checkout dev locally, then print a full success summary including branch rules, security artifacts, and the workflow guide. Offer to open repo in browser.

Security Hardening

  • gitleaks pre-commit hook
  • Comprehensive .gitignore
  • .gitattributes (eol=lf)
  • History scan on existing repos

Branching Model

  • main = protected, 1 reviewer
  • dev = protected, self-merge OK
  • Feature → dev PR flow
  • dev → main for releases

Error Handling

  • Name taken → suggest alternative
  • 403/422 → manual setup steps
  • Uncommitted changes → warn & stop
  • Old git fallback for --initial-branch
