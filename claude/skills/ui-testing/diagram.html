UI Verification

Catch invisible buttons, broken layouts, and contrast failures. Run these checks after creating any new UI components.

Pre-Flight Checklist — Before Shipping Any UI

Visibility

  • All buttons have visible bg OR border
  • No text same color as background
  • All text meets 4.5:1 contrast
  • Ghost buttons have visible borders

Touch / Click Targets

  • All buttons minimum 44px height
  • Icon buttons 44x44px minimum
  • Adequate spacing between clickables

States

  • Hover states visible (web)
  • Pressed states visible (mobile)
  • Focus rings on keyboard nav
  • Disabled: opacity 0.5
  • Loading states show indicators

Dark Mode + Responsive

  • Text readable on dark backgrounds
  • Borders visible in dark mode
  • No gray-400 on dark backgrounds
  • No horizontal scroll at 320px
  • Readable at all breakpoints

Common Fixes
Problem

Invisible Button

<button className="text-gray-500">Click</button>

Fix

Add Background or Border

className="bg-gray-100 text-gray-900 px-4 py-3 rounded-lg"

Problem

Low Contrast Text

<p className="text-gray-400">Secondary text</p>

Fix

Darken the Text

<p className="text-gray-600">Secondary text</p>

Problem

Missing Focus Ring

className="outline-none" (no ring)

Fix

Add Visible Focus Ring

outline-none focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2

Problem

Small Touch Target

className="p-1 text-sm" — less than 44px

Fix

Minimum 44px

className="w-11 h-11 flex items-center justify-center"

ESLint jsx-a11y

Automated accessibility linting via eslint-plugin-jsx-a11y. Catches missing ARIA roles, labels, and structural issues at write time.

npm install -D eslint-plugin-jsx-a11y // .eslintrc { "extends": [ "plugin:jsx-a11y/recommended" ] }

Playwright + axe-core

Full accessibility audit via @axe-core/playwright. Runs against live rendered pages in CI. Catches contrast, landmark, and ARIA violations.

import AxeBuilder from '@axe-core/playwright'; test('no a11y violations', async ({ page }) => { await page.goto('/'); const results = await new AxeBuilder({ page }) .analyze(); expect(results.violations).toEqual([]); });
