01

Component Architecture

High Impact
Anti-pattern — Boolean Props
<Modal
  hasHeader={true}
  hasFooter={false}
  isLarge={true}
  showCloseButton={true}
/>
→
Composition — Compound Components
<Modal>
  <Modal.Header>...</Modal.Header>
  <Modal.Body>...</Modal.Body>
  {/* No footer = no footer */}
</Modal>
architecture-avoid-boolean-props
Every boolean prop is a variation that should be a composition. Add subcomponents or variant exports instead of isLarge, hasHeader, showClose.
architecture-compound-components
Structure complex components as a family: Modal, Modal.Header, Modal.Body, Modal.Footer. Share context internally, expose clean API externally.
02

State Management

Medium Impact
state-decouple-implementation
The Provider is the only component that knows how state is managed. Components below consume context without caring about useState vs useReducer vs external store.
state-context-interface
Define a generic interface with three buckets: state (data), actions (mutations), meta (loading/error). This enables dependency injection and testing.
state-lift-state
Move state into provider components so siblings can access it. Don't thread state down through props — lift it to the nearest common provider ancestor.
03

Implementation Patterns

Medium Impact
Render Props — avoid
<List
  renderItem={(item) => (
    <Item data={item} />
  )}
/>
→
Children — prefer
<List>
  {items.map(item => (
    <Item key={item.id} data={item} />
  ))}
</List>
patterns-explicit-variants
Create <PrimaryButton> and <SecondaryButton> instead of <Button variant="primary">. Explicit components are self-documenting and tree-shakeable.
patterns-children-over-render-props
Use children for composition instead of renderX props. Children compose naturally, colocate with usage, and don't require prop-drilling custom renderers.
04

React 19 APIs

Medium Impact
⚠ React 19+ only — skip if using React 18 or earlier
react19-no-forwardref
Don't use forwardRef — React 19 passes refs as regular props. Use use() hook instead of useContext() for cleaner context consumption with conditional usage.
