homeradar-pre-pr

Pre-PR Gate

three self-gates · all must pass · no exceptions

Input — touched stacks

API uv / pytest / ruff / mypy
Extension npm test / type-check
Web npm test / e2e / lint / build

Gates — sequential, each must pass before proceeding

GATE 01
Test Gate
  • T API: uv run pytest
  • T Extension: npm test
  • T Web: npm test
  • T Web E2E: npm run e2e
  • ! Only run suites for stacks you touched
GATE 02
Quality Gate
  • Q API: ruff check src + mypy src
  • Q Extension: npm run type-check
  • Q Web: npm run lint + npm run build
  • ! Zero new errors allowed — existing baseline is acceptable
GATE 03
Spec Gate
  • S Every implementation commit has a Refs: line
  • S Refs count = total implementation commits
  • S No console.log introduced in .ts/.tsx/.py
  • S PR description includes Spec Coverage table (CAP-ID → ACs → test)

✓ All three gates pass
PR may be opened. The PR description must include a Spec Coverage table mapping each CAP-ID to its acceptance criteria rows and test names.
✕ Any gate fails
PR is blocked. Fix the failing gate, re-run, and verify before opening. A PR submitted with any failed gate will be rejected at review.
Hard rule: Gates are sequential — Gate 2 only runs after Gate 1 passes; Gate 3 only runs after Gate 2 passes. Run only the suites for stacks you touched. Check commits with git log origin/dev..HEAD and diff with git diff origin/dev..HEAD.
