PostHog Analytics

posthog-analytics skill — event tracking, feature flags, funnels, dashboards

Philosophy
Measure what matters, not everything.
Analytics should answer: Are users getting value? (activation, retention) — Where do users struggle? (funnels, drop-offs) — What features drive engagement? (feature usage) — Is the product growing? (acquisition, referrals). Track what informs decisions.
Installation by Platform
Next.js (App Router)
npm install posthog-js
Init in lib/posthog.ts, wrap with PostHogProvider, manual $pageview on pathname change.
React (Vite / CRA)
npm install posthog-js
Import PostHogProvider from posthog-js/react. Wrap <App /> at root.
Python (FastAPI)
pip install posthog
lru_cache() singleton. posthog.capture() + posthog.identify().
Node.js (Express)
npm install posthog-node
new PostHog(API_KEY). Flush on SIGTERM with posthog.shutdown().

Event naming: [object]_[action] in snake_case past tense — e.g. user_signed_up, subscription_upgraded. Never "click", "ButtonClick", "user signup".
User Identification
Identify on Signup / Login
Call posthog.identify(userId, { email, name, plan, created_at }). Reset on logout with posthog.reset() to clear identity.
Update Properties
Use posthog.capture('$set', { $set: { plan: 'pro' } }) when plan or role changes.
Standard Properties
Always track: email, name, created_at, plan, onboarding_completed, company_id (B2B), user_role.
Feature Flags & A/B Testing
Client-side Flag
useFeatureFlagEnabled('flag-name') hook returns boolean. useFeatureFlagPayload() for JSON config payloads.
Server-side (Next.js)
posthog.getAllFlags(userId) in Server Component. await posthog.shutdown() after use.
A/B Exposure Tracking
Always capture experiment_viewed with { experiment, variant } on mount to record exposure.

Privacy & Compliance
  • GDPR — opt_out_capturing_by_default: true. Explicit opt_in_capturing().
  • DNT — set respect_dnt: true in init config
  • Never track — passwords, credit cards, SSNs, full address
  • Sanitize props — filter keys containing "password", "token", "secret"
Testing Analytics
  • Dev mode — posthog.debug() + opt_out_capturing()
  • E2E (Playwright) — mock window.posthog in addInitScript, capture to window.capturedEvents[]
  • Unit test — assert event names and properties with expect(events).toContainEqual()
MCP workflow: Use dashboards-get-all → dashboard-create → query-run (test) → insight-create-from-query → add-insight-to-dashboard. Always create funnels: signup → onboarding → first value action.
