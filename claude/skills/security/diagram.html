Security is Non-Negotiable

OWASP security patterns, secrets management, and automated security testing. Every project must pass security checks before merge. Assume all input is malicious.

01 — OWASP Top 10 Key Patterns
A01 · A03
SQL Injection
Never concatenate user input into queries. Use parameterized queries or ORM.
db.query('SELECT * FROM users WHERE id = $1', [userId])
A03
XSS Prevention
Sanitize user HTML with DOMPurify. Avoid dangerouslySetInnerHTML. React escapes by default.
DOMPurify.sanitize(userInput)
A02
Crypto Failures
Use bcrypt (12+ rounds) for passwords. SHA1/MD5 are broken. Short JWT expiry.
bcrypt.hash(pass, 12)
A07
Auth Failures
Rate limit auth endpoints. Rotate session tokens. Check all routes require auth.
rateLimit({ max: 5/min })
A01
Path Traversal
Use path.basename() to strip ../. Verify final path is within allowed directory.
path.basename(req.params.filename)
A05
Security Headers
Use helmet middleware. Configure CSP directives. Enforce HTTPS with HSTS.
app.use(helmet({ ... }))
02 — Client-Exposed vs Server-Only Env Vars
Framework Client-Exposed Prefix Server-Only Risk if misused
Vite VITE_* No prefix API keys in browser bundle
Next.js NEXT_PUBLIC_* No prefix API keys in browser bundle
CRA REACT_APP_* N/A (no server) Everything exposed
Rule: NEVER put secrets (API keys, service role keys, DB passwords) in client-exposed prefixes. Verify in devtools → Sources that secrets aren't in the bundle.
03 — Patterns Comparison
❌ Insecure Patterns
Hardcoded API key: const key = "sk-proj-xxx"
SQL concat: `SELECT * WHERE id = ${id}`
Weak hash: MD5 or SHA1 for passwords
JWT no expiry or expiry > 24h
No rate limiting on /api/auth
Sensitive data in logs
CORS origin set to * in production
✅ Secure Patterns
Env var: process.env.API_KEY + validate at startup
Parameterized: query('...WHERE id=$1', [id])
bcrypt with 12+ salt rounds
JWT expiry: 15m access, 7d refresh
Rate limit: 5 req/min on auth endpoints
Log only userId, email — redact tokens
CORS restricted to known origins list
04 — Pre-Release Security Checklist

🔑 Secrets & Environment

No secrets in code (run detect-secrets)
.env files in .gitignore
.env.example with all required vars
Environment validated at startup (Zod)

🛡 Input & Database

All API inputs validated with schema
File uploads restricted by type & size
Parameterized queries only
Path traversal prevented

🔐 Auth & Headers

Passwords hashed with bcrypt (12+)
Rate limiting on auth endpoints
Security headers enabled (helmet)
CORS restricted to known origins

📦 Dependencies & Logs

npm audit / safety check passes
No known vulnerabilities in deps
No secrets in logs
No PII in logs (or properly masked)
