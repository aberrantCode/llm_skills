Middleware Layer
auth.ts
requireAuth
middleware/auth.ts
Extracts Bearer token, calls verifyToken() via Supabase, attaches req.user or returns 401.
validate.ts
validate(schema)
middleware/validate.ts
Wraps any Zod schema into Express middleware. Parses req.body and returns 400 with error.errors on failure.
error.ts
errorHandler
middleware/error.ts
Catches AppError (custom status codes) and generic errors. Last middleware in the chain.

const envSchema = z.object({
  PORT: z.string().default('3000'),
  SUPABASE_URL: z.string().url(),
  SUPABASE_ANON_KEY: z.string(),
  SUPABASE_SERVICE_ROLE_KEY: z.string(),

  envSchema.parse(process.env);
supabase.ts — two clients
// Anon key — respects RLS
export const supabase =
  createClient(url, anonKey);

// Admin — bypasses RLS
export const supabaseAdmin =
  createClient(url, serviceKey, {
    auth: {
      autoRefreshToken: false,
      persistSession: false
    }
  });
Hono Alternative (Edge-ready)
const app = new Hono();
app.use('/*', cors());

// Public route
app.get('/posts', async (c) => {
  const result = await db.select().from(posts)
    .where(eq(posts.published, true));
  return c.json(result);
});

// Auth middleware for protected routes
app.use('/api/*', async (c, next) => {
  const auth = c.req.header('Authorization');
  if (!auth?.startsWith('Bearer '))
    return c.json({ error: 'Unauthorized' }, 401);
  await next();
});
Anti-Patterns
✗
Using Supabase client for DB queries — use Drizzle; Supabase client is only for JWT verification and storage
✗
Sync JWT validation — verifyToken() must be async; Supabase auth calls are network requests
✗
No input validation — always use Zod middleware before route handlers; never trust req.body
✗
Missing error handling — use centralized errorHandler middleware; wrap all async handlers with try/catch
✗
Service key in responses — never include supabaseAdmin credentials in JSON responses or logs
✗
Blocking the event loop — keep all I/O async; sync database calls stall Node's event loop
