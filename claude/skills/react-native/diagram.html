Platform.select() — use sparingly
For simple differences, use Platform.select({ ios: ..., android: ... }) inside StyleSheet.create(). For complex differences, create separate Component.ios.tsx and Component.android.tsx files.

Component Patterns

Key Rules

Pressable over Touchable
Use Pressable instead of TouchableOpacity. Supports onPress, disabled, and style callbacks.
StyleSheet.create()
Never use inline style objects. Always define via StyleSheet.create() for performance and type safety.
Screens = Presentation Only
HomeScreen.tsx is pure UI. All logic lives in useHome.ts hook. Hooks are independently testable.
Explicit Props Interface
Always define ItemCardProps interface, even for simple components. No implicit any props.
Anti-Patterns

What to Avoid

✕ Inline styles
Use StyleSheet.create
✕ Logic in render
Extract to hooks
✕ Anonymous functions in props
Use useCallback
✕ Index as key in lists
Use stable IDs
✕ Direct state mutation
Always use setter
✕ Business logic in UI
Keep core/ React-free
