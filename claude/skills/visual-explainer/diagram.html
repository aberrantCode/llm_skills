skill · visual-explainer

Visual
Explainer

html diagrams · self-contained · always open in browser

Generate beautiful, self-contained HTML pages that visually explain systems, code changes, plans, and data. Triggered explicitly when a user asks for a diagram or overview — and proactively when about to render an ASCII table with 4+ rows or 3+ columns. Never fall back to ASCII art when this skill is loaded.
workflow phases
1
Think
Commit to a direction before writing HTML. Who is looking? What diagram type? Which aesthetic? Pick one and hold it. Constrained aesthetics (Blueprint, Editorial, Paper/ink) prevent generic output. The swap test: if a generic dark theme would be indistinguishable, push further.
2
Structure
Read the reference template before generating — every time. Choose rendering approach based on diagram type. Mermaid for graphs, flows, sequences, state machines. CSS Grid for text-heavy architecture cards. <table> for data, comparisons, audits.
3
Style
Distinctive font pairing — never Inter or Roboto as body. Forbidden accents: indigo/violet, fuchsia, cyan+magenta+pink. No gradient text on headings. No glowing animated shadows. Vary the palette every generation. Typography is the diagram.
4
Deliver
Write to ~/.agent/diagrams/. Open in browser immediately. Tell the user the file path. Run the squint test and swap test before shipping.

rendering decision guide
Diagram type Approach Reason
Flowchart / pipeline Mermaid graph TD/LR Automatic node positioning & edge routing
Sequence diagram Mermaid sequenceDiagram Lifelines, messages, activation boxes need auto layout
State machine Mermaid stateDiagram-v2 or flowchart State transitions with labeled edges; use flowchart if labels have special chars
ER / schema Mermaid erDiagram Relationship lines need auto-routing between many entities
Architecture (text-heavy) CSS Grid cards Rich card content (descriptions, code, lists) needs CSS control
Architecture (topology) Mermaid graph Visible connections between components need edge routing
Data table / comparison <table> element Semantic markup, accessibility, copy-paste behavior — not CSS grid
Timeline / roadmap CSS (central line + cards) Simple linear layout, no edge routing needed
Dashboard / metrics CSS Grid + Chart.js Card grid with embedded charts — Mermaid doesn't do dashboards well

constrained aesthetics (prefer these)
Blueprint
Deep slate/blue, grid background, monospace labels, precise borders. Technical drawing feel.
Paper / Ink
Warm cream #faf7f5, terracotta + sage accents, informal and handcrafted feel.
Editorial
Serif headline, generous whitespace, muted earth tones or deep navy + gold.
Terminal Mono
Green/amber on near-black, monospace everything, CRT glow optional.
IDE Theme
Commit to a real named theme: Dracula, Nord, Catppuccin, Gruvbox, Solarized. Not approximations.
Data Dense
Small type, tight spacing, maximum information, muted colors. No decoration.
slop signals — forbidden patterns
✕Inter or Roboto as --font-body
✕Indigo/violet accents #8b5cf6 #7c3aed
✕Gradient text on headings (background-clip:text)
✕Glowing animated box-shadows on cards
✕Cyan + magenta + pink neon combination
✕Perfectly uniform cards — no visual hierarchy
✕Three-dot code block chrome (red/yellow/green)
✕Neon Dashboard aesthetic — always generic
