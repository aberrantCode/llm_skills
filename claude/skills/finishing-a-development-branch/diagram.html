Skill Reference

Finishing a
Development Branch

superpowers:finishing-a-development-branch

Verifies tests pass, then presents exactly 4 structured options for completing feature work. Executes the chosen path and cleans up the worktree appropriately — never guesses what to do next.

Process Overview
Step 1
Verify Tests
Run full test suite. If any fail, STOP here.
Gate
Step 2
Determine Base Branch
Find the branch this feature diverged from.
Detect
Step 3
Present Options
Show exactly 4 options — no more, no less.
Required
Step 4
Execute Choice
Run the selected workflow path.
Action
Step 5
Cleanup Worktree
Remove for opts 1 & 4 only.
Conditional
Step 1 Decision Gate
?
Do all tests pass?
✓ Yes — continue to Step 2
✗ No — STOP. Show failures. Cannot proceed until tests pass.
Step 3 — The 4 Options (always presented in this order)
1
Merge Locally
  • Checkout <base-branch>
  • git pull
  • git merge <feature>
  • Run tests on merged result
  • If pass → git branch -d <feature>
Worktree: removed
2
Push & Create PR
  • git push -u origin <feature>
  • gh pr create with Summary + Test Plan
  • Branch stays open for review
Worktree: preserved
3
Keep As-Is
  • Report branch name
  • Report worktree path
  • No changes made
Worktree: preserved
4
Discard
  • Show branch + commits to be deleted
  • Require user to type discard
  • git branch -D <feature>
⚠ Typed confirmation required
Worktree: removed

Key Rules
🚫
Never skip test verification
Tests must pass before presenting options. Merging broken code or creating a failing PR are both blocked outcomes.
🚫
Never ask open-ended questions
"What should I do next?" is ambiguous. Always present exactly the 4 structured options — no deviation.
🚫
Never auto-cleanup for Option 2/3
Worktree stays alive for PR review (Option 2) and deferred work (Option 3). Only remove for 1 and 4.
✓
Always verify post-merge tests
After merging locally (Option 1), run the test suite again on the merged result before deleting the feature branch.
✓
Always get typed confirmation
Option 4 requires the user to type the exact word discard — prevents accidental deletion.
✓
Announce the skill at start
Say "I'm using the finishing-a-development-branch skill to complete this work" before beginning.
