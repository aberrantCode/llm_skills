Claude / Anthropic Integration
Basic Claude Bot
Use @anthropic-ai/sdk directly. Maintain per-user conversation history with Map<userId, MessageParam[]>. Cap at 20 messages.
Claude with Tool Use (Agent Loop)
Define tools[] (search KB, create task, get calendar). Run agent loop: call Claude → if stop_reason === "tool_use" → execute → continue until text response.
Teams SDK AI / OpenAI
ActionPlanner Setup
Configure OpenAIModel → PromptManager → ActionPlanner. Prompts in src/prompts/chat/ with config.json + skprompt.txt.
RAG with Azure AI Search
Query vector index → inject context into Claude prompt. Use semanticConfiguration + queryType: "semantic" for top-K retrieval.
Adaptive Cards
Welcome Card
Sent on membersAdded. Greets user by name. Action buttons: Get Started, View Help. Use CardFactory.adaptiveCard().
AI Response Card
Shows question + answer + optional sources. Actions: 👍 Helpful, 👎 Not Helpful, Ask Follow-up. Submit actions return { action, value }.
Form / Task Card
Input.Text (multi-line), Input.ChoiceSet (priority), Input.Date (due date). Required fields with validation. Cancel + Submit actions.
version: 1.5 schema: adaptivecards.io Validate at adaptivecards.io/designer Handle via adaptiveCard/action event

Deployment Options
  • Azure App Service — az webapp create + Bot Channels Registration
  • Teams Toolkit — teamsfx provision → deploy → publish
  • Docker — node:18-alpine, expose port 3978
  • Managed Identity — use for Azure resources, no secrets in env
Testing & Local Dev
  • ngrok tunnel — expose localhost:3978 for manifest webhook
  • Teams Toolkit debug — teamsfx preview --local
  • TestAdapter — unit test bot with botbuilder
  • Bot Framework Emulator — local bot testing without Teams
Security checklist: Validate all incoming messages — use App-Only auth for Graph API — never log sensitive user data — implement rate limiting — use managed identity in Azure — rotate secrets regularly — enable audit logging.
