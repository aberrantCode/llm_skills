Iterative Development

ralph wiggum loops · self-referential tdd iteration · clear completion criteria

Core Philosophy
Iteration > Perfection
Don't aim for perfect on first try. Let the loop refine the work. Each iteration builds on previous attempts visible in files and git history.
Failures Are Data
Failed tests, lint errors, type mismatches are signals. Use them to guide the next iteration — not reasons to panic or quit.
Clear Completion Criteria
Define exactly what "done" looks like. Tests passing. Coverage met. Lint clean. No ambiguity about when to stop iterating.

How the Loop Works
START
/ralph-loop
"Your task prompt"
--max-iterations N
→
WORK
Claude works
Reads files, writes code,
runs tests
→
EXIT?
Claude attempts exit
Outputs completion
or partial work
→
HOOK
Stop hook fires
Blocks exit, feeds
SAME PROMPT back
↓ Claude sees modified files + git history from before — iterates and improves ↓
Key insight: The prompt never changes. Only the file state evolves. Claude reads its own previous work and refines it. Loop continues until <promise>DONE</promise> is detected OR max iterations reached.

Good For (Clear Success Criteria)
  • TDD feature development — tests provide clear pass/fail
  • Bug fixes — reproduction → test → fix cycle
  • Refactoring — tests ensure no regressions
  • API development — each endpoint independently testable
  • Greenfield projects — iterate without human oversight
  • Getting tests to pass — clear success criteria
Not Good For (No Clear "Done")
  • Subjective design decisions — no objective metric
  • UI/UX work — requires human judgment
  • One-shot operations — no need for iteration
  • Unclear requirements — will loop indefinitely
  • Production debugging — needs human oversight
Required Prompt Elements
1. Clear Completion Criteria

### Completion Criteria
- [ ] All tests passing
- [ ] Coverage >= 80%
- [ ] Lint clean

### Exit Condition
<promise>DONE</promise>
2. Always Set Max Iterations

❌ /ralph-loop "task"
# runs forever!

✅ /ralph-loop "task"
  --max-iterations 30
  --completion-promise "DONE"
3. TDD in Every Prompt

### TDD Workflow
1. Write failing tests
2. Verify tests FAIL
3. Implement feature
4. Verify tests PASS
5. Run lint + typecheck
6. Repeat if failures
Anti-Patterns
No max-iterations (runs forever) Vague completion criteria No TDD in prompt Lying to exit (false promise) Manual intervention mid-loop Skipping test verification Continuing on access/env errors
