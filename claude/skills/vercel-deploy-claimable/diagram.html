Vercel Deploy

Deploy any project to Vercel instantly — no authentication required. Returns a live Preview URL and a Claim URL to transfer the deployment to your account.

1
Package the Project
Creates a deployment tarball from the project directory, automatically excluding node_modules and .git. Accepts a directory path or an existing .tgz file.
path argument (optional) defaults to cwd
2
Detect Framework
Auto-detects the framework from package.json. 30+ frameworks supported. For projects without package.json, framework is set to null (static HTML). Single non-index HTML files are auto-renamed to ensure root serving.
React / Next.js / Remix Vue / Nuxt SvelteKit Astro / Vite static HTML
3
Upload & Deploy
Uploads the tarball to the Vercel deployment service. No account or authentication token required — deployments are anonymous and claimable.
bash deploy.sh [path] no auth needed
4
Return Both URLs
Always present both URLs to the user. The Preview URL is immediately live. The Claim URL transfers the deployment to a Vercel account.
PREVIEW URL
https://skill-deploy-abc123.vercel.app
Live immediately. Accessible to anyone with the link. No account required to view.
CLAIM URL
https://vercel.com/claim-deployment?code=…
Transfers the deployment and project to your Vercel account for full management.

Network Egress Restriction
If deployment fails with a network restriction error (common on claude.ai), instruct the user to go to claude.ai/settings/capabilities and add *.vercel.com to allowed domains, then retry.
